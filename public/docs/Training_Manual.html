1. Introduction to the System

1.1 Purpose

The SDBIP & IDP Management System is a comprehensive web-based platform designed to help South African municipalities manage their Service Delivery and Budget Implementation Plan (SDBIP) and Integrated Development Plan (IDP) processes effectively.

1.2 Key Features

Module Description
Dashboard Executive overview with KPIs, charts, and performance summaries
IDP Strategic Objectives Manage integrated development plan strategic objectives
SDBIP & KPIs Service delivery budget implementation with KPI tracking
Quarterly Assessment Multi-level performance assessment workflow
Proof of Evidence Document upload and verification for KPI achievements
Budget Management Budget projections and capital projects tracking
Mayoral Imbizo Community engagement and action item tracking
Mayoral Lekgotla Priority management and IDP amendments
Reports & Analytics Comprehensive reporting with AI-powered insights

1.3 User Roles

Role Responsibilities Access Level
Administrator Full system configuration, user management, all modules Full Access
Director Directorate oversight, KPI approval, manager review Directorate Level
Manager KPI entry, self-assessment, team management Department Level
Independent Assessor Final assessment review and verification Assessment Module
User View dashboards and reports Read Only

2. Getting Started

2.1 Logging In

1
Open your web browser and navigate to the system URL provided by your administrator.
2
Enter your username (usually your email address) and password.
3
Click the "Login" button to access the system.
Note: If LDAP/Active Directory integration is enabled, use your network credentials to log in.

2.2 Navigating the Interface

The system uses a sidebar navigation layout with the following components:

  • Sidebar (Left): Main navigation menu with all available modules
  • Top Bar: Breadcrumb navigation, notifications, and user menu
  • Content Area: Main working area displaying the selected module
  • Footer: Current financial year and quarter information

2.3 Understanding Financial Years and Quarters

South African municipalities operate on a July-June financial year cycle:

Quarter Period Assessment Deadline
Q1 July - September October 15
Q2 October - December January 15
Q3 January - March April 15
Q4 April - June July 15

3. Dashboard Overview

3.1 Executive Dashboard

The dashboard provides a real-time overview of organizational performance:

Performance Cards

  • Total KPIs: Number of active key performance indicators
  • Overall Achievement: Percentage of KPIs meeting targets
  • Pending Assessments: Assessments awaiting completion
  • Budget Utilization: Current expenditure vs. budget

Charts and Visualizations

  • Performance Trend: Quarterly performance over time
  • Directorate Comparison: Performance by directorate
  • KPI Status Distribution: Achieved, partially achieved, not achieved
  • Budget Allocation: Budget distribution across projects
Tip: Click on any chart element to drill down into detailed data.

4. IDP Strategic Objectives

4.1 Understanding IDP Objectives

IDP Strategic Objectives represent the long-term developmental goals of the municipality. They are organized hierarchically:

  • National Development Plan (NDP) Pillars: Aligned with national priorities
  • Provincial Growth Strategy: Provincial development frameworks
  • Municipal Strategic Objectives: Local priorities and goals

4.2 Creating an IDP Objective

1
Navigate to IDP > Objectives from the sidebar menu.
2
Click "Add New Objective" button.
3
Fill in the required fields:
  • Objective Code (e.g., SO1, SO2)
  • Objective Title
  • Description
  • NDP Pillar alignment
  • Responsible Directorate
4
Click "Save" to create the objective.

4.3 Linking KPIs to Objectives

Each KPI should be linked to at least one IDP objective to ensure alignment between operational activities and strategic goals.

5. SDBIP & KPI Management

5.1 Understanding SDBIP

The Service Delivery and Budget Implementation Plan (SDBIP) translates the IDP and budget into measurable outputs and outcomes through Key Performance Indicators (KPIs).

5.2 KPI Structure

Field Description Example
KPI Code Unique identifier FIN-001
KPI Name Descriptive title Revenue Collection Rate
Baseline Starting measurement 85%
Annual Target Year-end goal 95%
Quarterly Targets Q1, Q2, Q3, Q4 targets 88%, 90%, 93%, 95%
Unit of Measure How it's measured Percentage

5.3 Creating a KPI

1
Navigate to SDBIP > KPIs from the sidebar.
2
Click "Create New KPI" button.
3
Complete all required fields including quarterly targets.
4
Link the KPI to an IDP objective.
5
Assign responsible directorate and manager.
6
Click "Save" to create the KPI.
Important: Ensure quarterly targets progressively lead to the annual target. The system will validate target consistency.

6. Quarterly Assessment Process

6.1 Assessment Workflow

The system implements a three-tier assessment process:

Level Assessor Weight Purpose
Level 1 Self-Assessment 20% Manager reports own performance
Level 2 Manager Review 40% Director validates and adjusts
Level 3 Independent Assessment 40% External verification

6.2 Completing Self-Assessment

1
Navigate to Assessment > Self Assessment.
2
Select the KPI to assess.
3
Enter the actual achieved value for the quarter.
4
Provide comments explaining performance.
5
Upload Proof of Evidence documents.
6
Submit for manager review.

6.3 Rating Scale

Rating Description Achievement
5 Outstanding 150%+ of target
4 Exceeds Expectations 115-149% of target
3 Meets Expectations 100-114% of target
2 Below Expectations 50-99% of target
1 Unacceptable Below 50% of target

7. Proof of Evidence (POE)

7.1 POE Requirements

Proof of Evidence documents substantiate KPI achievements and are required for audit compliance.

Acceptable Document Types

  • PDF documents
  • Microsoft Word documents (.doc, .docx)
  • Microsoft Excel spreadsheets (.xls, .xlsx)
  • Images (.jpg, .jpeg, .png)
  • Maximum file size: 10MB per document

7.2 Uploading POE

1
Navigate to POE > Upload from the sidebar.
2
Select the quarterly assessment to attach POE.
3
Click "Choose File" and select your document.
4
Add a description of the document.
5
Click "Upload" to submit.

7.3 POE Review Process

Managers can accept or reject POE submissions:

  • Accept: POE is verified and linked to assessment
  • Reject: POE is returned with comments for resubmission
Note: Users have 7 days to resubmit rejected POE documents.

8. Budget Management

8.1 Budget Projections

Track and manage budget allocations across directorates and projects.

Budget Categories

  • Operating Budget: Day-to-day operational expenses
  • Capital Budget: Infrastructure and asset investments
  • Conditional Grants: Ring-fenced government allocations

8.2 Capital Projects

Manage capital project lifecycle from planning to completion:

Status Description
Planning Project in design/planning phase
Procurement Tender and procurement processes
Implementation Active construction/delivery
Completed Project finished and handed over
On Hold Temporarily suspended

9. Mayoral IDP Imbizo

9.1 Overview

The Mayoral IDP Imbizo module facilitates community engagement sessions with integrated livestreaming, comment extraction, and action item tracking.

9.2 Creating an Imbizo Session

1
Navigate to Imbizo > Create from the sidebar.
2
Enter session details (ward, venue, date, time).
3
Configure livestream platforms (YouTube, Facebook, Twitter/X, Municipal Stream).
4
Save the session.

9.3 During the Imbizo

  • Comments Extraction: Automatically pull comments from connected social media platforms
  • Minutes Capture: Record opening remarks, discussion points, concerns, and commitments
  • Action Items: Create and assign action items to directorates

9.4 Action Item Assignment

Action items can be assigned to specific:

  • Directorates
  • Departments
  • Individual employees

10. Mayoral Lekgotla

10.1 Purpose

The Lekgotla module enables management of IDP priority changes resulting from strategic planning sessions.

10.2 Priority Change Types

Type Description
Add New Priority Introduce a new IDP priority from Imbizo commitments
Modify Existing Adjust budget or scope of existing priority
Discard Priority Remove priority and free up budget

10.3 Comparison View

Compare IDP priorities before and after Lekgotla sessions to track changes and their impact on the overall plan.

13. Troubleshooting & FAQ

13.1 Common Issues

Cannot log in

  • Verify username and password are correct
  • Check if account is active
  • Clear browser cache and cookies
  • Contact administrator for password reset

POE upload fails

  • Check file size (max 10MB)
  • Verify file type is supported
  • Ensure stable internet connection

Charts not loading

  • Refresh the page
  • Disable browser extensions
  • Try a different browser

13.2 Frequently Asked Questions

Q: How do I reset my password?

A: Contact your system administrator to request a password reset.

Q: Can I access the system on mobile devices?

A: Yes, the system is responsive and works on tablets and smartphones.

Q: How long is data retained?

A: Data is retained for the current and previous 5 financial years.

Q: Who can see my assessments?

A: Your assessments are visible to your manager, directors, and administrators.
